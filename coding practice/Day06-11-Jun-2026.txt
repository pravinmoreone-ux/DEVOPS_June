## 🔹 GitLab CI — Debugging
You will learn how to **handle job failures and exit codes**.

## Defines the order of pipeline stages.
stages:
  - build
  - test

##If the build stage fails, the test stage will not run.
build: ### Creates a job named build.
  stage: build
  image: maven:3.8.1-jdk-17
  script:
    - git clone https://github.com/Pathnex/sample-java-app.git
    - cd sample-java-app
    - mvn clean compile || { echo "Compile failed"; exit 1; }

test: ### create a job test
  stage: test
  image: maven:3.8.1-jdk-17
  script:
    - cd sample-java-app
    - mvn test || { echo "Tests failed"; exit 1; }


    ##vArtifacts are files or directories generated by a job and stored by GitLab so they can be downloaded or used by subsequent jobs and stages in the pipeline.

    Q: Why can the test job fail even if the build job succeeds?

Answer:
Each GitLab CI job runs in a separate container/environment. Files created in one job are not automatically available in the next job unless they are passed using artifacts or recreated in the new job.
